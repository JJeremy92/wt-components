## Best practices

### Labels

Keep Button Labels concise, clear, and action-oriented. Use verbs like "Submit," "Save," and "Cancel". Avoid vague Labels like "Click here." Consistent Labels improve the user experience.

#### Leading icons

        Use Leading Icons to visually support an action, and provide additional context to what is happening. For example, adding a ‚Äú+‚Äù to an ‚ÄúAdd new X‚Äù Button helps provide additional context to the action.

#### Trailing icons

Use Trailing Icons to better support the user in understanding the UI process the action is about to prompt. For example, using an arrow can indicate that a Button will open a dropdown or a new window.

## Button variants

### Primary

Primary Buttons are reserved for the main call-to-actions of a page or panel. Primary Buttons have the highest visual impact of the Button variants and, therefore, communicate actions that move the workflow forward.

<ul>
    <li>
        Use a single primary action per page to keep the purpose of your next logical step clear. This is what you want
        your user to click in a normal scenario.
    </li>
</ul>

<Canvas></Canvas>

### Default

Ghost Buttons or Text Buttons are typically the lowest priority actions in a design, especially when presenting multiple options on a page or panel.

<ul>
    <li>Use the isError variant for destructive actions such as "discard" or "delete."</li>
</ul>

### Ghost

Ghost Buttons or Text Buttons are typically the lowest priority actions in a design, especially when presenting multiple options on a page or panel.

<ul>
    <li>It can be used to provide multiple supplementary actions if there is no clear primary or secondary action.</li>
    <li>Best used for non-critical choices in a workflow, such as cancelling or subscribe</li>
</ul>
